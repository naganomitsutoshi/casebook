> 依頼元: CEO（2026-07-04 指示「SPEC-v3 をブラッシュアップしたものを Fable 5 のみで実装」）
> 実装体制: Claude Code（Fable 5）直接実装（v3 と同体制、Codex 不使用）
> 基点コード: v3（コミット `fc049f2`）。**データモデル・純ロジック・保存キーは一切変更しない**

## 0. 位置づけ

- 主環境はスマホ、副環境はPC。v4 は見た目と操作の最適化のみで、機能・書き出し形式は v3 と完全互換
- 実機の既存データ（`casebook:v2`）はそのまま読める。`<script id="logic">` は無変更

## 1. 不変の制約（SPEC-v3 §1 と同一）

1. 単一 HTML・外部ライブラリなし・外部送信なし
2. PII 入力欄を作らない
3. ユーザー入力は `esc()` 経由で innerHTML へ
4. テキスト入力は debounce 保存、入力中に全画面 `render()` を呼ばない
5. モバイル優先・タップターゲット 44px 以上
6. ストレージキー `casebook:v2` 維持
7. 純ロジックは `<script id="logic">` 分離、Node からテスト可能

## 2. v4 の変更内容

### 2.1 モバイル操作系（≤640px）

- **下部ナビ追加**：一覧／新規／書き出し（件数バッジ）／ゴミ箱。`env(safe-area-inset-bottom)` 対応
- トップバーの操作ボタンはモバイルでは非表示（下部ナビに集約）、トップバー自体は非スティッキー化して縦領域を節約
- **症例タブ（今日・過去ログ・経過表）をスティッキー化**：スクロール中もタブ切替可能
- トーストは下部ナビと重ならない位置へ

### 2.2 ダークモード

- `prefers-color-scheme: dark` で自動切替（夜勤・当直想定）。全色を CSS 変数化し、ハードコード色を排除
- `color-scheme: light dark` で date 入力等のネイティブ UI も追従。`theme-color` メタをライト／ダーク両方定義

### 2.3 配置の最適化

- 今日のビューの並びを作業優先に変更：**今日やる → 待ち → もしもプラン → ひっかかり → 問題リスト**（書き出しの節順は不変）
- 一覧の症例カードに患者サマリの2行プレビューを追加
- メトリクスを実数に合わせ3カラム化（旧: 4カラム定義に3項目）

### 2.4 入力効率

- todo／待ち／ひっかかり／もしもプランの追加欄で **Enter キー追加**＋追加後のフォーカス維持（連続入力）
- 新規症例モーダルの「一言」欄も Enter で作成
- 行アイテムを1行レイアウトに圧縮：チェック(44px)＋テキスト＋×削除(44px)。旧2段構成比で高さ約半分
- 写真追加はネイティブ file input をボタン風ラベルに変更

### 2.5 経過表

- 薬剤行はセル背景の**帯表示**（内服=青系／注射=橙系）、開始日・終了日にタグ表示
- イベント点は種別の頭文字入り（検/手/処/治/そ）
- 凡例を表の下に追加。今日列の強調・sticky 先頭列・横スクロールは維持

### 2.6 バグ修正（v3 からの持ち越し）

- **過去ログタブの入力 ID 重複**：`todoInput` 等が日数分重複し、過去日への追加が別の日の入力欄を拾っていた → `${kind}Input-${date}` に変更
- **一覧フィルタチップの選択状態が無表示**：`.chip.on` の CSS が存在しなかった → 追加
- 症例画面の保存/削除ボタンがモバイルで消えないよう、非表示化はトップバー内の `.header-actions` に限定

### 2.7 変更してはいけないもの

- `<script id="logic">` 全体（書き出し形式・繰越・後方互換・ROUTINE 10項目）
- Store アダプタ／IndexedDB 写真管理／ゴミ箱 30日 purge／手動保存

## 3. テスト

- `tests/verify-v2.js`：無改修で全通過を維持（ロジック無変更の裏取り）
- `tests/smoke-render.js`（v4 新設）：DOM スタブ上でメイン script を実行し、一覧／今日／過去ログ／経過表／ゴミ箱の描画関数が例外なく期待要素（tabs-bar・band 帯・日付付き入力 ID・パネル並び）を含む HTML を返すか検証
